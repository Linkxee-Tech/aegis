# Aegis — API Reference

Base URL: `http://localhost:8000/api` (configurable via `API_PREFIX`)

Interactive OpenAPI docs are auto-generated by FastAPI at `/docs` and `/redoc`
once the server is running — this file covers the same surface in plain prose
for quick reference.

## Authentication

If `AUTH_ENABLED=true`, every REST request must include either:

- `Authorization: Bearer <api-key>`
- `X-API-Key: <api-key>`

WebSocket connections accept the same API key via a `token` query parameter.

Roles are keyed by the configured environment variables:

- `VIEWER_API_KEYS` can read dashboard data
- `OPERATOR_API_KEYS` can read data and approve/reject/simulate incidents
- `ADMIN_API_KEYS` can access every endpoint

## GET /health

Returns the system health summary shown in the dashboard's status strip.

```json
{
  "allAgentsOperational": true,
  "lastIncidentAgo": "2m ago",
  "activeIncidentCount": 1,
  "uptimePercentage": 99.94
}
```

## GET /agents

Returns current status for all five agents (Detective, Diagnostician,
Remediation, Reporter — Memory is tracked internally but not surfaced as a
status card today).

```json
[
  {
    "id": "detective",
    "name": "Detective",
    "role": "Monitors logs & metrics for anomalies",
    "model": "Qwen-Flash",
    "status": "active",
    "metricLabel": "alerts today",
    "metricValue": 12,
    "color": "detective"
  }
]
```

`status` is one of `active`, `idle`, `thinking`, `error`, `awaiting_approval`.

## GET /incidents

Returns all known incidents, live ones first. Falls back to demo data if the
Coordinator has no incidents yet (e.g. right after a fresh deploy).

## GET /incidents/{incident_id}

Returns a single incident by id, including its full timeline and remediation
steps. 404s if the id doesn't match a live or demo incident.

## POST /incidents/{incident_id}/approve

The human checkpoint. Approves the pending remediation plan and triggers
execution against live infrastructure via `RemediationAgent.execute_steps`.

Response: `{"success": true}`. Returns `409` if there's no pending approval for
that incident (already resolved, rejected, or the approval window expired —
see `APPROVAL_TIMEOUT_SECONDS`).

## POST /incidents/{incident_id}/reject

Rejects the pending plan. Body is optional:

```json
{ "reason": "Wrong root cause — investigate cache layer instead" }
```

Response: `{"success": true}`. Same 409 semantics as approve.

## GET /memory

Returns all patterns the Memory Agent has learned, for the Memory dashboard
page — incident title, root cause, fix applied, occurrence count, confidence,
and whether the pattern is currently eligible for auto-apply.

## GET /reports

Returns all generated incident reports.

## GET /reports/{report_id}/download

Streams a PDF export of the report. Not yet implemented in this build — see
the `backend/api/routes.py` download endpoint and `backend/services/report_export.py`.

## WebSocket /ws

Streams live events as the agent pipeline runs. Each message:

```json
{
  "type": "timeline_event",
  "payload": { "incidentId": "INC-2026-0614", "event": { "...": "..." } }
}
```

`type` is one of `incident_created`, `incident_updated`, `agent_status_changed`,
`timeline_event`, `approval_required`, `report_generated` — matching
`frontend/src/hooks/useWebSocket.ts`'s expected event shape.
